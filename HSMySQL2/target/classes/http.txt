HTTP 方法（GET、POST、PUT、PATCH、DELETE）時，我們可以透過一個送禮物的生活情境來說明，並結合 URL 和 form-data 的概念：
GET - 瀏覽禮物選項
透過這個送禮物的比喻，我們可以更直觀地理解 HTTP 方法在 Web 應用中如何運作，以及 URL 和表單數據（form-data）如何在這些請求中發揮作用。

GET 請求：
URL: https://example.com/gifts
動作: 透過這個 URL，你「獲取」（GET）了一個禮物清單的頁面，上面列出了所有可選的禮物。
回應: 服務器會回應一個包含禮物清單的 HTML 頁面或 JSON 資料，讓你可以看到所有可選的禮物。

URL: https://example.com/gifts?id=123 <- id 表示請求參數
URL: https://example.com/gifts/123 <- 這裡的 123 是路徑
動作: 透過這個 URL，你「獲取」（GET）了一個特定禮物的詳細資訊，例如 ID 為 1 的禮物。
回應: 服務器會回應一個包含禮物詳細資訊的 HTML 頁面或 JSON 資料，讓你可以看到這份禮物的描述、圖片等。

------------------------------------------------------------------------------------------------------------

POST - 選擇並送出禮物
決定好要送什麼禮物後，你會填寫一個表單，選擇一個禮物並附上一段祝福的話，然後提交（POST 請求）這個表單來完成送禮：
URL: https://example.com/send-gift
form-data:
    giftId: 123（你選擇的禮物 ID）
    message: 祝你生日快樂！（你想要附上的祝福語）
動作: 透過 POST 請求，你「提交」了一份表單，表單中包含了禮物的選擇和你的祝福語，服務器接收到這份表單後，會處理送禮的流程。
回應: 服務器可能會回應一個確認訊息(HTML 頁面或 JSON 資料)，告訴你已經成功送出了禮物。
------------------------------------------------------------------------------------------------------------

PUT - 更換禮物
如果你後來決定要送另一份禮物，你可能需要填寫一個新表單來更新你的選擇（PUT 請求）：

URL: https://example.com/update-gift
form-data:
    giftId: 456（你新選擇的禮物 ID）
動作: PUT 請求「替換」了原先的禮物選擇。服務器接收到這份表單後，會更新你的送禮選擇。
回應: 服務器可能會回應一個確認訊息(HTML 頁面或 JSON 資料)，告訴你已經成功更換了禮物。

------------------------------------------------------------------------------------------------------------

PATCH - 修改禮物附言

如果你只是想更改附在禮物上的祝福語，而不改變禮物本身，你可以提交一個 PATCH 請求：

URL: https://example.com/update-message
form-data:
    message: 祝你新年快樂！（你新的祝福語）
動作: PATCH 請求「部分修改」了禮物的附加資訊，即祝福語(HTML 頁面或 JSON 資料)，而不影響禮物的選擇。

------------------------------------------------------------------------------------------------------------

DELETE - 取消送禮

如果因某種原因你決定取消送禮，你會發出一個 DELETE 請求來撤銷送禮：

URL: https://example.com/cancel-gift
form-data: 可能不需要或只需提供禮物的 ID。
動作: 透過 DELETE 請求，你「取消」了之前的送禮計畫，服務器接收到請求後，會撤銷之前的送禮操作。
回應: 服務器可能會回應一個確認訊息(HTML 頁面或 JSON 資料)，告訴你已經成功取消了送禮。
